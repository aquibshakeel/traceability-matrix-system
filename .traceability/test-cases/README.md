# Test Cases Directory

This directory contains test cases in a two-folder architecture:

## Directory Structure

### ğŸ“ baseline/
**Purpose:** QA-managed test scenarios (version controlled)
- Simple JSON files with business test scenarios
- Written by QA team in plain English
- Version controlled (committed to git)
- Updated manually by QA

**Example:**
```json
{
  "testCases": [
    {
      "testId": "CUST-001",
      "description": "Create customer with valid email returns 201",
      "api": "/api/customers",
      "method": "POST",
      "expectedStatus": 201
    }
  ]
}
```

### ğŸ“ ai_cases/
**Purpose:** AI-generated test cases (regenerated fresh)
- Comprehensive test scenarios generated by Claude AI
- Regenerated on every run
- NOT version controlled (in .gitignore)
- Includes happy path, edge cases, error scenarios

**Example:**
```json
{
  "testId": "POST_/api/customers_001",
  "description": "Create customer with valid data returns 201",
  "testType": "happy_path",
  "priority": "P1",
  "request": {
    "body": { "name": "John Doe", "email": "john@example.com" }
  },
  "expectedResponse": {
    "statusCode": 201
  }
}
```

### ğŸ“ reports/
**Purpose:** Delta analysis reports
- Shows differences between baseline and AI-generated cases
- Identifies new test ideas from AI
- Flags outdated baseline cases
- Provides QA action items
- NOT version controlled

## Workflow

1. **QA writes baseline** - Create simple test scenarios in `baseline/`
2. **Run AI generation** - Execute `npm run generate`
3. **Review reports** - Check `reports/` for delta analysis
4. **Update baseline** - Add good AI suggestions to `baseline/`
5. **Repeat** - Keep baseline up to date

## Commands

```bash
# Generate test cases with AI
npm run generate

# Clean AI cases and reports (keeps baseline)
npm run clean
```
