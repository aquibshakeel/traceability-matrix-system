Phase 1: AI Test Generation

  • Swagger Parsing: Reads OpenAPI/Swagger specifications
  • AI Generation: Claude generates comprehensive test scenarios
  • Categorization: Organizes into happy_case, edge_case, error_case, security
  • Output: Saves AI suggestions to review and use

Phase 2: Coverage Analysis

  • Load Baseline: Reads QA-defined expected scenarios
  • Parse Tests: Discovers all unit tests in codebase
  • AI Matching: Claude intelligently matches scenarios to tests
  • Gap Analysis: Identifies what's covered, partial, or missing

Four Test Categories

Category Purpose Examples
happy_case Normal, successful flows Valid input returns 200, Data saved correctly
edge_case Boundary conditions Empty list, Max items, Min/max values
error_case Error handling Invalid input returns 400, Not found returns 404
security Authentication & authorization No token returns 401, Forbidden returns 403
