üîç How Does The System Work?

The Core Question

"How does the system know which unit test belongs to which business scenario?"

Answer: AI-Powered Text Matching

The system uses Claude AI with natural language understanding to match test descriptions to baseline scenarios - not manual fuzzy matching or pattern matching.

Three-Step Process

  1. QA Defines Scenarios - In simple YAML format using "When...then..." statements
  2. System Parses Tests - Extracts unit tests with @DisplayName annotations
  3. AI Matches & Analyzes - Claude AI determines coverage status with HIGH/MEDIUM/LOW confidence
